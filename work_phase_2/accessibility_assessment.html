Accessibility

Overview of WCAG Principles

This website is developed in alignment with the WCAG 2.2 guidelines to ensure accessibility, inclusivity, and usability for users with diverse needs. It focuses on the following principles:

  • Perceivable: Content must be presented in ways that users can perceive.
  • Operable: Interface components must be operable via different input methods.
  • Understandable: Information and UI must be easy to understand and use.
  • Robust: Content must be compatible with assistive technologies.

Relevant Success Criteria & Client/User Needs

Based on the client brief and user interviews (Aisha, David, Lily), we identified the following relevant success criteria:

1.1.1 Alternative Text for Non-Text Content

Helps users like Aisha, who benefit from visual cues due to limited English proficiency. Alternative text ensures icons and images are meaningful and accessible.

1.3.2 Meaningful Sequence

Ensures that content is presented in a logical order for screen readers, supporting Aisha's need for simplified, non-overwhelming content.

1.4.3 Perceivable Text Contrast

Ensures readability across different devices, especially helpful for users like Lily who access the site on mobile with limited lighting conditions.

1.4.4 Resize Text without Loss

Supports users with limited literacy or vision by allowing zooming without breaking the layout, improving usability across all demographics.

2.1.1 Keyboard Navigability

Essential for mobile navigation and low-tech users like Aisha, who use only a phone and need intuitive tab-based access.

2.4.7 Focus Visible

Highlights navigation focus for users with limited technical literacy, enhancing clarity when browsing with a keyboard or screen reader.

3.1.5 Reading Level

Ensures that text is understandable for non-native speakers like Aisha and Lily by using plain language and avoiding jargon.

3.2.3 Consistent Navigation

Reduces confusion for users like David who value professionalism and clarity in layout and interaction.

3.2.4 Consistent Identification (Layout & UI Behaviour)

Ensures familiar interactions throughout the site, which is critical for trust-building as mentioned by both Lily and David.

3.3.2 Labels & Instructions

Supports input clarity for newcomers like Aisha, ensuring forms and UI elements are labeled with clear instructions to avoid confusion.

Rationale

The selected criteria reflect the mission of the Connect Communities Initiative by promoting inclusion, clarity, and accessibility. Each guideline directly supports the needs of specific users:

  • Aisha: Benefits from clean UI, visual cues, text alternatives, and low reading level content.
  • David: Needs professional, consistent design with intuitive navigation and accessible forms.
  • Lily: Requires a mobile-friendly design with high contrast, readable layout, and simple, trustable content.

Meeting these criteria enables the website to be accessible, trustworthy, and usable for the targeted community groups.
